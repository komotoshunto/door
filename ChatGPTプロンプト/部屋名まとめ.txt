1. 上記の部屋名称のリストから、「名称のバリエーション」だけを代表名称にまとめてください。
   ここで「まとめる」とは、次の条件を満たす複数の名称を、
   共通する「ベース名（代表名称）」のキーに統合し、
   元の名称をすべて値のリストとして格納することを意味します。

2. 「まとめてよい」名称のバリエーション（= 同じベース名とみなせる条件）は次の通りです。

   2.1 まず、各名称を「ベース名」+「末尾のバリエーション部分」に分解できるか判定してください。
       ここで「ベース名」とは、名称の先頭側の部分であり、
       「末尾のバリエーション部分」を取り除いた残りの部分です。

   2.2 「末尾のバリエーション部分」として認められるパターンは、次のいずれかに「完全に」一致する場合のみです。
       それ以外の末尾は、バリエーション部分とはみなさないでください。

       (A) 室名や番号:
           - 半角数字／全角数字のみ
             例: "101", "２", "10" など
           - ハイフン "-" に続く「数字や数字の組み合わせ」だけ
             例: "-1", "-2-3", "-10-2" など
           - アンダースコア "_" に続く「数字だけ」
             例: "_1", "_2", "_10" など

       (B) 記号付きの短い英字:
           - ハイフン "-" に続く 1〜2 文字程度の英字だけ
             例: "-A", "-B", "-C" など

       (C) 短い英字・漢字・カナ:
           - 末尾の 1 文字だけがバリエーションである場合
             例: "北", "南", "A", "B" など

       ※ 重要: 「_Main」「_Backup」のように、
          アンダースコアの後ろが英字列の場合（"_Main", "_Backup" など）は、
          上記のどのパターンにも一致しないため、
          バリエーション部分として扱ってはいけません。

   2.3 複数の名称について、「ベース名」が同一であり、
       かつ「末尾のバリエーション部分」が 2 件以上存在すると判定できる場合、
       そのベース名でまとめてください。

       代表名称（ベース名）は、この「末尾のバリエーション部分」を取り除いた文字列とします。

       例（あくまでパターンの例であり、実際の入力名称とは限りません）:
       - 「教室101」「教室102」「教室103」
         → ベース名は「教室」
         → 出力: "教室": ["教室101", "教室102", "教室103"]

       - 「Office-1」「Office-2」「Office-10」
         → ベース名は「Office」
         → 出力: "Office": ["Office-1", "Office-2", "Office-10"]

       - 「Zone_1」「Zone_2」「Zone_5」
         → ベース名は「Zone」
         → 出力: "Zone": ["Zone_1", "Zone_2", "Zone_5"]

       - 「研究区画1」「研究区画2」
         → ベース名は「研究区画」
         → 出力: "研究区画": ["研究区画1", "研究区画2"]

       - 「LabA-1」「LabA-2」「LabA-10」
         → ベース名は「LabA」
         → 出力: "LabA": ["LabA-1", "LabA-2", "LabA-10"]

       - 「研究室A」「研究室B」「研究室C」
         → ベース名は「研究室」
         → 出力: "研究室": ["研究室A", "研究室B", "研究室C"]

       - 「保管室西」「保管室東」
         → ベース名は「保管室」
         → 出力: "保管室": ["保管室西", "保管室東"]

       - 「Block-A」「Block-B」
         → ベース名は「Block」
         → 出力: "Block": ["Block-A", "Block-B"]

       - 「ブース1」「ブース01」「ブース２」
         → ベース名は「ブース」
         → 出力: "ブース": ["ブース1", "ブース01", "ブース２"]

       - 「ServerZone_1」「ServerZone_2」
         → ベース名は「ServerZone」
         → 出力: "ServerZone": ["ServerZone_1", "ServerZone_2"]

3. 重要: ベース名にまとめるためには、
   「同じベース名 + 許可されたパターンの末尾のバリエーション部分」の組み合わせが
   2つ以上存在することが必須条件です。

   3.1 同じベース名に属すると判定できる名称が 1 つしかない場合、
       その名称はベース名にまとめず、「元の名称そのまま」をキーとして扱ってください。

       例（パターンの例）:
       - 入力に「ギャラリー1」しかない場合
         → 出力: "ギャラリー1": ["ギャラリー1"]

       - 入力に「待合室2」しかない場合
         → 出力: "待合室2": ["待合室2"]

       - 入力に「打合せスペース3」しかない場合
         → 出力: "打合せスペース3": ["打合せスペース3"]

4. 「まとめない」ケース（共通部分があっても代表名称に統合しないケース）は次の通りです。

   4.1 「ベース名」が完全に一致していない場合は、共通する語が含まれていてもまとめないでください。

       例（パターンの例）:
       - 「中央テラス」と「南テラス広場」
         → 共通部分「テラス」が含まれますが、ベース名は同じではありません。
         → 出力:
            "中央テラス": ["中央テラス"],
            "南テラス広場": ["南テラス広場"]

       - 「男子トイレ」「女子トイレ」「多目的トイレ」
         → 「トイレ」を含みますが、ベース名が同じとはみなさず、それぞれ別とします。
         → 出力:
            "男子トイレ": ["男子トイレ"],
            "女子トイレ": ["女子トイレ"],
            "多目的トイレ": ["多目的トイレ"]

       - 「イベントスペースA」と「展示スペースB」
         → 「スペース」を含みますが、ベース名は異なります。
         → 出力:
            "イベントスペースA": ["イベントスペースA"],
            "展示スペースB": ["展示スペースB"]

       - 「案内デスク」と「総合サービスカウンター」
         → 「カウンター」等の共通語があっても、ベース名が同じではないのでまとめません。

   4.2 アンダースコア "_" の後ろが英字列の場合は、
       その末尾部分は「許可されたバリエーションパターン」には該当しません。
       したがって、その名称はベース名にまとめず、別のキーとして扱ってください。

       例（パターンの例）:
       - 「ServerZone_1」「ServerZone_2」「ServerZone_Main」「ServerZone_Backup」がある場合
         → "_1" と "_2" はバリエーション部分として認められますが、
            "_Main" と "_Backup" は数字だけではないため、
            バリエーション部分として扱ってはいけません。
         → 正しい出力:
            "ServerZone": ["ServerZone_1", "ServerZone_2"],
            "ServerZone_Main": ["ServerZone_Main"],
            "ServerZone_Backup": ["ServerZone_Backup"]

   4.3 同じベース名であっても、
       末尾の部分が上記 2.2 のどのパターンにも「完全一致」しない場合は、
       その名称はバリエーションとして扱わず、個別のキーにしてかまいません。

5. 入力される部屋名称には、連番などの規則性がない場合があります。
   上記のルールおよび例から、
   「どのようなときにまとめてよいか」「どのようなときにまとめてはいけないか」
   を学習し、同様のパターンに従って処理してください。
   特定の名称だけに特別な処理をするのではなく、
   「ベース名が一致しているか」「末尾が許可されたバリエーションパターンか」
   というルールに基づいて判断してください。

6. 重要: 用途・機能ごと（トイレ・倉庫・廊下など）にカテゴリ名を新しく作って
   グルーピングする処理は一切行わないでください。
   例えば「トイレ・洗面」「倉庫」などの抽象的なカテゴリキーを新たに作成してはいけません。
   あくまで「名称のフォーマットや番号／短いバリエーション違い」を
   代表名称（ベース名）にまとめるだけにしてください。

7. 出力形式:
   出力は必ず JSON オブジェクトとし、キーが「代表名称（ベース名）または元の名称」、
   値が「その代表名称に属する元の名称のリスト」となるようにしてください。

8. 出力を生成する前に、このプロンプト内のすべての条件（1〜7および9〜10）に違反していないかを必ず内部的に確認してください。
   もし違反が見つかった場合は、その出力は破棄し、条件を満たすように再度 JSON を生成し直してください。
   ただし、この確認ややり直しを行った事実や内容を、最終的な出力（JSON）の中に一切含めてはいけません。

9. このプロンプトは Python プログラムの途中にあることを想定しています。

10. このプロンプトによって出力されたものがそのまま Python プログラムに入力されるため、
   必ず JSON だけを出力してください。
   文字列メッセージやコメント、説明文は一切出力しないでください。

11. 「以下のJSONをコピーしてください」などの説明文や余分なテキストは絶対に出力しないでください。
